How It Works
1

Broken Code

A Rust project with compilation errors is loaded — missing dependencies, wrong features, syntax issues.

2

Read Diagnostics

The agent sees cargo check output with error codes like E0432, E0277, helping it reason about the fix.

3

Apply Fix

The agent edits Cargo.toml or src/main.rs — one file per step, or both at once with both_files.

4

Earn Reward

Partial reward for reducing errors, full reward for a clean build. Regression is penalized.

Why Oxidizer

🎯 Real Compiler, Not Simulated

Every step runs the actual Rust compiler via cargo check. The agent sees the same error codes and diagnostics a human developer would see. No mocks, no approximations.

📈 Partial Reward Shaping

Unlike binary pass/fail environments, Oxidizer rewards partial progress. Reducing errors from 5 to 2 earns reward proportional to the improvement — providing gradient signal for RL training.

🛡 Security Hardened

OWASP Top 10:2025 compliant. Optional API key auth, Pydantic input validation, opaque error IDs, compiler output sanitization for LLM prompts (ASI04 defense).

🔧 Rich Action Space

Three action types: single-file edit, both_files for atomic multi-file repair, and dry_run for planning steps that re-read diagnostics without committing changes.

📊 Structured Info Dict

Each step returns error counts, warning counts, regression flags, and progress tracking — giving RL trainers richer signal beyond just the scalar reward.

🤖 MCP Tool Server

Full JSON-RPC 2.0 endpoint at /mcp with tools: get_compiler_errors, get_task_state, list_tasks. Agents connecting via MCP get a proper tool interface.
